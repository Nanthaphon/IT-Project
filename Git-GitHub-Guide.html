1. ภาพรวมการทำงาน

เข้าใจการทำงานก่อนลงมือทำ

การสลับเครื่องทำงานคือการใช้ GitHub เป็นสะพานกลาง โดยคุณ "ฝาก" โค้ดไว้บน GitHub แล้วเครื่องอื่นก็ "หยิบ" โค้ดมาทำต่อ

💻
เครื่อง A
ทำงาน + แก้โค้ด
→
☁️
GitHub
เก็บโค้ดกลาง
→
🖥️
เครื่อง B
ดึงมาทำต่อ

A เครื่อง A → GitHub (เรียกว่า PUSH)

คำสั่งทำอะไร
git add .เลือกไฟล์ที่จะส่งขึ้น
git commit -m "..."สร้างจุดบันทึก + เขียนข้อความบอกว่าทำอะไร
git push origin mainส่งขึ้น GitHub

B GitHub → เครื่อง B (เรียกว่า PULL หรือ CLONE)

คำสั่งใช้เมื่อ
git clone <url>ครั้งแรกที่เครื่องนั้นยังไม่มีโค้ด
git pull origin mainมีโค้ดอยู่แล้ว แค่อัปเดตล่าสุด
!
กฎทอง: ก่อนปิดเครื่องทุกครั้ง ต้อง push ขึ้น GitHub เสมอ ไม่งั้นเครื่องอื่นจะดึงโค้ดล่าสุดของคุณไม่ได้

2. เครื่องนี้: เอาโค้ดขึ้น GitHub

ทำเมื่อ: แก้โค้ดเสร็จ และต้องการเก็บไว้บน GitHub

1
เปิด PowerShell ไปที่โฟลเดอร์โปรเจกต์
เปิด PowerShell แล้วใช้ cd ไปที่โฟลเดอร์
cd "C:\Users\IT\Desktop\IT Project"
2
เช็คว่ามีไฟล์อะไรเปลี่ยนบ้าง
ดูว่าแก้ไขไฟล์อะไรไปบ้าง
git status

→ ถ้าเห็น "modified:" หรือ "Untracked files:" แสดงว่ามีของให้ push

3
เพิ่มไฟล์ทั้งหมดเข้า staging
บอก Git ว่า "ฉันจะ commit ไฟล์เหล่านี้"
git add .
4
Commit พร้อมข้อความ
เขียนสั้นๆ ว่าทำอะไรไป (ภายในเครื่องหมาย " ")
git commit -m "เพิ่มฟีเจอร์ KPI Dashboard"
5
Push ขึ้น GitHub
ส่งโค้ดขึ้นไปบน GitHub branch ชื่อ main
git push origin main
✓
สำเร็จเมื่อเห็น: main -> main ที่บรรทัดสุดท้าย

3. เครื่องใหม่: โหลดโค้ดมาครั้งแรก

ทำเมื่อ: เครื่องนั้นยังไม่เคยมีโค้ดเลย

i
ก่อนเริ่ม: เครื่องใหม่ต้องติดตั้ง Git และ Node.js ก่อน ดูได้ที่:
• Git → git-scm.com/downloads
• Node.js → nodejs.org (เลือก LTS)
1
เปิด PowerShell ไปที่ Desktop
หรือโฟลเดอร์ที่ต้องการเก็บโปรเจกต์
cd C:\Users\YourName\Desktop
2
Clone โปรเจกต์มาจาก GitHub
ดาวน์โหลดทั้ง repo มาที่เครื่อง (รันครั้งเดียวเท่านั้น)
git clone https://github.com/Nanthaphon/IT-Project.git
3
เข้าไปในโฟลเดอร์โปรเจกต์
โค้ดจริงๆ อยู่ใน subfolder ชื่อ it_asset_menagement
cd IT-Project\it_asset_menagement
4
ติดตั้ง dependencies
npm install จะดาวน์โหลด library ทั้งหมดที่โปรเจกต์ต้องการ (รันครั้งแรก รอ 1-3 นาที)
npm install
5
รัน Development Server
เปิดเว็บที่ http://localhost:5173 เพื่อทดสอบ
npm run dev
✓
เสร็จเรียบร้อย! ตอนนี้เครื่องใหม่พร้อมแก้โค้ดและ push ขึ้น GitHub ได้แล้ว

4. เครื่องที่มีโค้ดอยู่แล้ว: อัปเดตล่าสุด

ทำเมื่อ: เครื่องนั้นเคย clone แล้ว แค่อยากดึงโค้ดใหม่

1
เปิด PowerShell ไปที่โฟลเดอร์โปรเจกต์
โฟลเดอร์ที่เคย clone ไว้
cd C:\path\to\IT-Project
2
Pull โค้ดล่าสุดจาก GitHub
ดึงการเปลี่ยนแปลงใหม่ๆ มาทับโค้ดเก่า
git pull origin main
3
ติดตั้ง dependencies เพิ่ม (ถ้ามี)
ทำเฉพาะเมื่อ package.json มีการเปลี่ยนแปลง (มี library ใหม่)
cd it_asset_menagement npm install
4
รัน Development Server
พร้อมทำงานต่อแล้ว
npm run dev
!
ข้อควรระวัง: ก่อน pull ถ้าเครื่องนี้แก้ไฟล์ค้างไว้ (ยังไม่ commit) Git จะเตือน — ให้ git commit ก่อน หรือใช้ git stash เพื่อพักไว้ชั่วคราว

5. ปัญหาที่พบบ่อย + วิธีแก้

เจอ error ไม่ต้องตกใจ ดูตารางนี้

❌ "Please enter a commit message to explain why this merge is necessary"

เกิดเมื่อ pull แล้วโค้ดท้องถิ่นกับ remote ขัดกัน Git เปิด vim editor

# กด Esc แล้วพิมพ์: :wq

❌ "Your local changes would be overwritten by merge"

คุณแก้โค้ดค้างไว้ แต่ยังไม่ commit — Git ไม่ยอม pull ทับ

# ทางเลือกที่ 1: commit ก่อน git add . git commit -m "บันทึกชั่วคราว" git pull origin main # ทางเลือกที่ 2: stash (พักไว้ชั่วคราว) git stash git pull origin main git stash pop

❌ "Updates were rejected because the remote contains work that you do not have locally"

เครื่องอื่น push ขึ้นไปแล้ว — เครื่องนี้ต้อง pull ก่อน push ใหม่

git pull origin main git push origin main

❌ "fatal: not a git repository"

อยู่ผิดโฟลเดอร์ — Git command ทำงานได้เฉพาะโฟลเดอร์ที่มี .git ซ่อนอยู่

cd "C:\Users\IT\Desktop\IT Project" # กลับไปโฟลเดอร์โปรเจกต์

❌ "Permission denied (publickey)"

ใช้ SSH URL แต่ยังไม่ได้ตั้งค่า — ใช้ HTTPS URL แทน

git remote set-url origin https://github.com/Nanthaphon/IT-Project.git

6. คำสั่ง Git ที่ใช้บ่อย

เก็บไว้เป็น cheatsheet

คำสั่งความหมาย
git statusดูสถานะ มีไฟล์ไหนแก้ไป / staged / untracked
git add .เพิ่มทุกไฟล์ที่แก้เข้า staging
git add <file>เพิ่มเฉพาะไฟล์ที่ระบุ
git commit -m "msg"บันทึก commit พร้อมข้อความ
git push origin mainส่ง commit ขึ้น GitHub
git pull origin mainดึงโค้ดล่าสุดจาก GitHub
git clone <url>โหลด repo ทั้งหมดมาเครื่อง (ครั้งแรก)
git log --oneline -10ดูประวัติ commit 10 รายการล่าสุด
git diffดูว่าแก้อะไรไปบ้างเทียบกับ commit ก่อน
git stashพักการแก้ไขชั่วคราว (เก็บไว้ก่อน)
git stash popเอาการแก้ไขที่พักไว้กลับมา
git restore <file>คืนค่าไฟล์กลับเป็นแบบใน commit ล่าสุด
git branchดูว่าตอนนี้อยู่ branch อะไร

⚡ สรุปขั้นตอนสลับเครื่อง (ฉบับสั้นที่สุด)

A
ก่อนปิดเครื่องนี้:
git add . && git commit -m "ทำอะไรไปบ้าง" && git push origin main
B
เปิดเครื่องอีกตัวมาทำต่อ:
git pull origin main
!
ห้ามลืม! ก่อนเปลี่ยนเครื่อง — push ก่อนเสมอ ไม่งั้นอีกเครื่องจะเอาโค้ดล่าสุดของคุณไปต่อไม่ได้ และอาจเกิดความขัดแย้งของไฟล์ภายหลัง
